2. RINGKASAN REVISI & PERBAIKAN

Setelah menerima laporan bug dari hasil peer-testing, tim developer (pengembang) telah melakukan serangkaian perbaikan kode secara komprehensif pada aplikasi web Culturestone Indonesia. Fokus perbaikan mencakup aspek keamanan autentikasi, sanitasi input, validasi file upload, penguatan password policy, serta perbaikan kualitas tampilan pada panel admin dan footer.

Total 10 bug berhasil diidentifikasi dan diperbaiki, terdiri dari 2 bug Critical, 4 bug Major, dan 4 bug Minor. Seluruh perbaikan telah diverifikasi melalui pengujian unit (unit testing) dan pengujian integrasi (integration testing) menggunakan framework testing bawaan Go dengan database SQLite in-memory, menghasilkan 46 test case dengan pass rate 100%.

Perbaikan dilakukan pada 8 file kode sumber yang tersebar di layer backend (Go) dan frontend (React.js), dengan tidak merusak fungsionalitas fitur-fitur lain yang sebelumnya telah berjalan normal.

3. TABEL PELACAKAN & RESOLUSI BUG (BUG RESOLUTION TRACKING)

Menindaklanjuti temuan bug dari hasil pengujian, berikut adalah tindakan perbaikan yang telah dilakukan:

ID Bug Fitur / Modul Deskripsi Bug Awal Severity Status Akhir Solusi Perbaikan
BUG-01 Autentikasi Admin Halaman login admin menampilkan tombol “Daftar” yang memungkinkan siapapun mendaftarkan akun admin baru tanpa otorisasi apapun. Critical Fixed (1) Backend: Ditambahkan pengecekan jumlah admin di database sebelum registrasi. Jika sudah ada admin, endpoint mengembalikan HTTP 403 Forbidden. Route POST /api/register dihapus dari public route. (2) Frontend: Tombol “Daftar” dan seluruh form registrasi dihapus dari Login.jsx. (3) Auto-seed: Admin dibuat otomatis dari variabel ADMIN_USERNAME dan ADMIN_PASSWORD di file .env saat server pertama kali dijalankan melalui fungsi seedAdmin() di main.go.
BUG-02 Autentikasi Admin Response API register mengembalikan objek admin lengkap termasuk field Password berisi hash bcrypt, sehingga hash bocor ke client. Critical Fixed Ditambahkan tag json:"-" pada field Password di model Admin. Response register diubah hanya mengembalikan message dan username, tidak lagi mengirim objek admin.
BUG-03 Upload File Service upload file tidak memvalidasi jenis file. File berbahaya seperti .exe, .php, .sh dapat diunggah ke server. Major Fixed Diterapkan dua lapis validasi: (1) whitelist ekstensi file (.jpg, .jpeg, .png, .gif, .webp, .mp4, .mov, .avi), dan (2) validasi MIME type dari 512 byte pertama isi file menggunakan http.DetectContentType.
BUG-04 Form Kontak Publik Input dari form kontak (Name, Phone, Email, Message) langsung disimpan ke database tanpa sanitasi, memungkinkan serangan Stored XSS. Major Fixed Semua input disanitasi menggunakan html.EscapeString dari standard library Go sebelum disimpan ke database. Karakter berbahaya (<, >, ", ', &) di-escape secara otomatis.
BUG-05 Autentikasi Admin Tidak ada validasi kekuatan password saat registrasi admin. Admin dapat mendaftar dengan password sangat sederhana seperti "1" atau string kosong. Major Fixed Ditambahkan fungsi validatePassword() yang mewajibkan password minimal 8 karakter, mengandung minimal 1 huruf kapital, 1 huruf kecil, dan 1 angka.
BUG-06 Form Kontak Publik Input nomor HP tidak divalidasi sehingga menerima huruf dan karakter sembarang, menghasilkan data tidak valid di database. Minor Fixed Frontend: Input type="tel" dengan filter realtime via onChange hanya mengizinkan angka, +, -, spasi. Backend: Validasi regex ^[0-9+-s]{8,20}$ sebelum simpan ke DB.
BUG-07 Form Kontak Publik Input nama tidak divalidasi sehingga menerima angka dan karakter aneh (contoh: "12345", "!!!@@@") sebagai nilai nama. Minor Fixed Frontend: Filter realtime via onChange hanya mengizinkan huruf (termasuk unicode) dan spasi. Backend: Validasi regex ^[p{L}s]{2,100}$ sebelum simpan ke DB.
BUG-08 Dashboard Admin Dashboard admin menampilkan persentase hardcoded menyesatkan: "+5% bulan lalu", "+8% bulan lalu", "-3% minggu ini". Data ini tidak dihitung dari data real. Variabel dummyStats dan dummyRecentOrders tidak terpakai. Minor Fixed Dihapus variabel dummyStats dan dummyRecentOrders. Semua teks persentase hardcoded diganti dengan label netral "Data dari database". Statistik sepenuhnya berasal dari API GET /api/admin/stats.
BUG-09 Footer Publik Ikon Instagram dan Facebook di footer menggunakan href="#" sehingga klik tidak mengarahkan ke halaman sosial media Culturestone. Minor Fixed Ganti href="#" dengan URL sosial media aktif Culturestone. Ditambahkan target="_blank", rel="noopener noreferrer", dan aria-label untuk keamanan dan aksesibilitas.
BUG-10 Upload File URL file yang diunggah di-generate dengan format hardcoded http://localhost:8080/uploads/filename. Saat di-deploy ke production, semua URL gambar tidak dapat diakses. Major Fixed Base URL diambil dari environment variable BASE_URL di file .env. Jika kosong, fallback ke http://localhost:8080. Untuk production cukup ubah nilai BASE_URL tanpa menyentuh kode.
Catatan: Tidak ada bug dengan status Deferred atau Rejected. Seluruh 10 bug berhasil diperbaiki (status Fixed).
