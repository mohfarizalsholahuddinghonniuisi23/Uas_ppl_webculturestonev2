BAB I
PENDAHULUAN

1.1 Latar Belakang

Perkembangan teknologi digital telah mendorong berbagai sektor industri untuk memanfaatkan website sebagai media informasi, promosi, dan komunikasi dengan pelanggan. Website company profile tidak hanya berfungsi sebagai identitas digital perusahaan, tetapi juga menjadi sarana untuk memperkenalkan produk, layanan, serta nilai-nilai yang dimiliki kepada masyarakat luas.

Culturstone merupakan aplikasi web yang hadir sebagai media informasi resmi perusahaan yang bergerak di bidang batu alam. Website ini dibangun untuk memberikan informasi mengenai profil perusahaan, katalog produk batu alam, portofolio proyek, kategori produk, serta media komunikasi dengan pelanggan melalui fitur kontak dan testimoni.

Aplikasi web Culturstone dikembangkan menggunakan arsitektur fullstack dengan React.js (Vite) pada sisi frontend dan Go (Gin Framework) pada sisi backend, serta menggunakan SQLite sebagai database. Website ini dilengkapi dengan panel admin yang diamankan menggunakan JWT (JSON Web Token) untuk pengelolaan produk, kategori, portofolio, testimoni, dan pesan kontak.

Dalam pengembangan website, diperlukan proses pengujian perangkat lunak untuk memastikan bahwa seluruh fitur dan fungsi sistem berjalan dengan baik sesuai kebutuhan pengguna. Pengujian dilakukan untuk meminimalkan kesalahan sistem, meningkatkan kualitas website, serta memastikan integrasi antar halaman dan modul berjalan secara optimal.

Pada laporan ini dilakukan beberapa metode pengujian, yaitu Black Box Testing, Unit Testing, dan Integration Testing. Black Box Testing digunakan untuk menguji fungsi sistem berdasarkan input dan output tanpa melihat struktur kode program. Unit Testing dilakukan untuk menguji fungsi atau unit program secara individual, sedangkan Integration Testing dilakukan untuk memastikan seluruh modul website dapat saling terhubung dan bekerja dengan baik.

Melalui proses pengujian tersebut diharapkan aplikasi web Culturstone mampu memberikan pengalaman pengguna yang baik, memiliki performa yang stabil, serta mendukung aktivitas promosi dan penyebaran informasi perusahaan secara optimal.

1.2 Tujuan Pengujian

Tujuan dilakukannya pengujian pada aplikasi web Culturstone adalah sebagai berikut:

  1. Memastikan seluruh fitur website berjalan sesuai dengan kebutuhan sistem.
  2. Mengidentifikasi kesalahan atau bug pada website sebelum digunakan oleh pengguna.
  3. Memastikan fungsi setiap komponen program berjalan dengan baik melalui Unit Testing.
  4. Memastikan integrasi antar halaman dan modul website berjalan dengan benar melalui Integration Testing.
  5. Menjamin kualitas, stabilitas, dan keandalan website.
  6. Memastikan informasi perusahaan dan produk dapat ditampilkan dengan baik kepada pengguna.

1.3 Ruang Lingkup Pengujian

Ruang lingkup pengujian pada aplikasi web Culturstone meliputi beberapa bagian berikut:

  • Halaman Publik (Produk, Portofolio, Testimoni, Kategori, Kontak)
  • Sistem autentikasi admin (Register, Login, dan Proteksi JWT)
  • Fitur CRUD Produk pada panel admin
  • Fitur CRUD Kategori pada panel admin
  • Fitur CRUD Portofolio pada panel admin
  • Fitur CRUD Testimoni
  • Fitur Dashboard Admin (Statistik)
  • Fitur Kelola Pesan Masuk

BAB II
METODOLOGI PENGUJIAN

2.1 Metode Pengujian

Pengujian pada aplikasi web Culturstone dilakukan menggunakan tiga metode utama yang saling melengkapi untuk memastikan kualitas sistem secara menyeluruh.

2.1.1 Black Box Testing

Black Box Testing merupakan metode pengujian yang dilakukan dari perspektif pengguna akhir tanpa mengetahui struktur internal kode program. Pengujian ini dilakukan secara manual pada aplikasi web Culturstone yang mencakup beberapa modul berikut:

  • Halaman publik (Produk, Portofolio, Testimoni, Kategori)
  • Sistem autentikasi admin (Register, Login, dan Proteksi JWT)
  • Fitur CRUD Produk, Kategori, Portofolio, Testimoni pada panel admin
  • Dashboard Admin dan Kelola Pesan Masuk

Teknik pengujian yang digunakan meliputi:

  1. Equivalence Partitioning — Digunakan untuk membagi data input menjadi kelas valid dan tidak valid sehingga pengujian lebih efektif.
  2. Boundary Value Analysis — Digunakan untuk menguji batas minimum dan maksimum input.
  3. Error Guessing — Digunakan untuk menguji kemungkinan error berdasarkan pengalaman penguji.

Pembagian pengujian Black Box Testing:

  • Autentikasi & Tampilan Publik (F01–F05) : [Nama Anggota 1]
  • Kategori, Kontak & CRUD Produk (F06–F10) : [Nama Anggota 2]
  • CRUD Lanjutan, Dashboard & Keamanan (F11–F15) : [Nama Anggota 3]

2.1.2 Unit Testing

Unit Testing merupakan metode pengujian yang dilakukan terhadap unit atau komponen terkecil dari program secara terpisah untuk memastikan setiap fungsi bekerja dengan benar sesuai logika yang telah dirancang.

Pada aplikasi web Culturstone, Unit Testing dilakukan menggunakan Go testing (standard library) terhadap beberapa komponen berikut:

  • Controller Admin (AdminRegister, AdminLogin)
  • Controller Publik (GetProducts, GetPortfolios, GetTestimoni, GetCategories, GetProductCategories)
  • Controller CRUD (SaveProduct, SaveCategory, SavePortfolio, SaveTestimoni, PostContact)
  • Controller Admin (GetAdminStats, GetMessages, DeleteEntity)
  • JWT Token Utilities (GenerateToken, ExtractToken, TokenValid)

Pengujian ini bertujuan untuk memastikan bahwa method, validasi, konfigurasi, dan proses CRUD berjalan dengan baik secara individual.

Penanggungjawab: [Nama Anggota 2]

2.1.3 Integration Testing

Integration Testing merupakan metode pengujian yang dilakukan untuk memastikan integrasi antar modul atau komponen sistem dapat berjalan dengan baik. Pengujian dilakukan terhadap hubungan antara:

  • Router (Gin)
  • Controller
  • Database (GORM + SQLite)
  • Middleware (JWT)
  • HTTP Request/Response

Pada aplikasi web Culturstone, Integration Testing dilakukan menggunakan httptest.Server (Go standard library) untuk memastikan proses autentikasi, CRUD produk, CRUD kategori, CRUD portofolio, CRUD testimoni, manajemen pesan, dan proteksi JWT dapat berjalan secara terintegrasi.

Penanggungjawab: [Nama Anggota 1]

2.2 Tahapan Pengujian

  1. Analisis Kebutuhan Pengujian — Tahap awal dilakukan dengan mengidentifikasi fitur dan fungsi sistem yang akan diuji berdasarkan kebutuhan aplikasi.
  2. Penyusunan Test Case — Pada tahap ini dibuat skenario pengujian yang mencakup kondisi valid, invalid, serta kemungkinan terjadinya error pada sistem.
  3. Pelaksanaan Pengujian — Pengujian dilakukan menggunakan metode Black Box Testing, Unit Testing, dan Integration Testing.
  4. Evaluasi Hasil Pengujian — Hasil pengujian dianalisis untuk mengetahui apakah sistem berjalan sesuai harapan atau masih terdapat kesalahan.
  5. Dokumentasi Hasil Pengujian — Seluruh hasil pengujian dicatat dan disusun ke dalam laporan hasil pengujian perangkat lunak.

2.6 Skenario Pengujian

Skenario pengujian disusun berdasarkan kebutuhan sistem dan dibagi menjadi tiga jenis pengujian utama:

  1. Black Box Testing — Pengujian dilakukan pada fungsi web dari sisi pengguna dengan memeriksa kesesuaian output terhadap input yang diberikan menggunakan teknik Equivalence Partitioning, Boundary Value Analysis, dan Error Guessing.
  2. Unit Testing — Pengujian dilakukan terhadap method dan fungsi tertentu secara individual menggunakan Go testing standard library dengan database SQLite in-memory.
  3. Integration Testing — Pengujian dilakukan untuk memastikan komunikasi dan integrasi antar komponen sistem (Router → Middleware → Controller → Database → Response) berjalan dengan baik.

2.7 Kriteria Keberhasilan Pengujian

Pengujian dinyatakan berhasil apabila:

  • Sistem menghasilkan output sesuai dengan hasil yang diharapkan.
  • Tidak terjadi error atau crash pada sistem saat pengujian dilakukan.
  • Validasi input berjalan dengan benar.
  • Integrasi antar modul dapat berjalan tanpa kendala.
  • Seluruh test case menunjukkan status PASS.

3.3.2 Detail Hasil Unit Testing

Tidak ditemukan kegagalan dalam unit testing. Seluruh komponen berfungsi sesuai spesifikasi. Berikut catatan penting dari proses pengujian:

  • AdminRegister & AdminLogin: Proses register, login, validasi input, dan penanganan error berhasil diuji secara individual.

  • JWT Token Utilities: Fungsi GenerateToken, ExtractToken, dan TokenValid berhasil diverifikasi untuk semua skenario (valid, invalid, kosong).

4.1 Kesimpulan

Berdasarkan hasil pengujian perangkat lunak yang telah dilakukan pada aplikasi web Culturstone menggunakan metode Black Box Testing, Unit Testing, dan Integration Testing, dapat disimpulkan bahwa sistem telah berjalan dengan baik sesuai dengan kebutuhan pengguna dan perancangan sistem yang telah dibuat.

Hasil pengujian menunjukkan bahwa seluruh fitur utama web dapat berfungsi secara optimal, baik dari sisi pengguna umum maupun administrator sistem. Seluruh halaman publik seperti halaman produk, portofolio, testimoni, kategori, dan kontak berhasil ditampilkan dengan baik tanpa mengalami error. Selain itu, sistem autentikasi admin juga berhasil melakukan proses register, login, validasi, serta pengamanan halaman admin menggunakan JWT middleware dengan benar.

Pada pengujian CRUD, seluruh proses seperti tambah, edit, hapus produk/kategori/portofolio/testimoni, upload gambar, serta validasi form berhasil dijalankan sesuai dengan kebutuhan sistem. Sistem juga berhasil menangani kondisi invalid seperti input kosong, data duplikat, serta akses data yang tidak valid.

Pengujian Unit Testing menggunakan Go testing standard library menunjukkan bahwa seluruh method dan fungsi pada controller dan utilitas JWT dapat berjalan dengan baik secara individual. Seluruh 31 test case unit testing memperoleh hasil PASS dengan persentase keberhasilan sebesar 100%.

Pengujian Integration Testing juga menunjukkan bahwa integrasi antar komponen seperti Router, Controller, Database (GORM), Middleware (JWT), dan HTTP Request/Response dapat berjalan dengan baik tanpa terjadi kegagalan sistem. Seluruh 20 test case integration testing berhasil dijalankan dengan hasil PASS sebesar 100%.

Secara keseluruhan, hasil pengujian menunjukkan bahwa aplikasi web Culturstone memiliki kualitas sistem yang baik, stabil, dan layak digunakan sebagai media informasi dan promosi perusahaan berbasis web.

4.2 Saran

Meskipun sistem telah berjalan dengan baik, terdapat beberapa saran pengembangan yang dapat dilakukan untuk meningkatkan kualitas web di masa mendatang, antara lain:

  1. Menambahkan validasi client-side dan server-side pada form kontak (F08) yang saat ini menerima input kosong.
  2. Menambahkan fitur pencarian produk dan filter kategori yang lebih kompleks agar pengguna lebih mudah menemukan produk yang diinginkan.
  3. Menambahkan fitur keamanan tambahan seperti:
    • Proteksi CSRF lebih lanjut
    • Rate limiting login
    • Two-factor authentication
    • Enkripsi data sensitif
  4. Mengimplementasikan pengujian otomatis berbasis Continuous Integration (CI/CD) agar proses testing dapat berjalan otomatis setiap terjadi perubahan kode program.
  5. Mengembangkan tampilan website agar lebih responsif dan optimal pada berbagai ukuran perangkat mobile maupun tablet.
  6. Menambahkan lebih banyak test case untuk edge case (misalnya: upload file berukuran sangat besar, karakter spesial pada input).
  7. Menambahkan pengujian performa (Performance Testing) dan pengujian keamanan (Security Testing) untuk meningkatkan kualitas sistem secara keseluruhan.
  8. Mempertimbangkan implementasi automated end-to-end testing (misalnya dengan Selenium/Cypress) untuk melengkapi black-box testing manual.
